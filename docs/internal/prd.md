# Product Requirements Document: Travel Blog with Affiliate Monetization

**Version:** 1.0
**Date:** 2026-03-07
**Status:** Draft

## Executive Summary

A high-performance static travel site built with Astro, optimized for SEO and affiliate monetization. Content is structured JSON generated by an AI pipeline and consumed by purpose-built renderers вЂ” design and content are fully decoupled.

### Success Metrics

**Primary Goal:** Quality traffic with volume

**Performance Metrics:**
- **Lighthouse Score:** 95+ on all metrics (Performance, SEO, Accessibility, Best Practices)
- **Core Web Vitals:** Pass all metrics (LCP < 2.5s, FID < 100ms, CLS < 0.1)
- **Page Speed:** First Contentful Paint < 1.5s

**Traffic Metrics:**
- **Organic Traffic:** Primary growth indicator
- **Bounce Rate:** < 60% (indicates quality content)
- **Average Session Duration:** > 2 minutes
- **Pages Per Session:** > 1.5

**SEO Metrics:**
- **Indexed Pages:** 90%+ of published content
- **Keyword Rankings:** Track position for target keywords
- **Backlinks:** Quality over quantity, track referring domains
- **Domain Authority:** Monitor month-over-month growth

**Engagement Metrics:**
- **Affiliate Click-Through Rate:** > 3% of page views
- **Scroll Depth:** > 50% on average
- **Return Visitor Rate:** > 20%

**Content Metrics:**
- **Content Velocity:** Sustainable publishing cadence (quality > quantity)
- **Content Depth:** Average post word count 1,500-2,500 words
- **Content Freshness:** Regular updates to evergreen content

## Product Vision

### Problem Statement
WordPress and traditional CMS platforms introduce unnecessary overhead, security vulnerabilities, and performance penalties for content creators who want:
- Fast loading times (better SEO and user experience)
- Simple affiliate link integration
- Low/zero hosting costs
- Modern development workflow

### Solution
Static site generator (Astro) with a JSON-first content pipeline providing:
- AI-generated structured content at scale
- Zero-JavaScript static performance
- Free hosting on GitHub Pages
- Built-in SEO and analytics
- GTM-based affiliate click tracking

### 2. Affiliate Link Management

#### 2.1 Core Principle: Affiliate Metadata in JSON, Logic in Renderers
**Priority:** P0

Affiliate data lives in JSON content files. Renderers own all affiliate presentation logic вЂ” `rel` attributes, disclosure rendering. Click tracking is handled entirely by GTM вЂ” no custom tracking code in the codebase.

- Updating affiliate tracking в†’ GTM config change only, zero code touched
- Redesigning product cards в†’ change the renderer, zero content files touched
- Adding a new affiliate partner в†’ add URL to JSON content, done

**Affiliate metadata schema (in JSON content):**
```typescript
interface AffiliateProduct {
  name: string;
  affiliate_url: string;            // Full URL with tracking tag
  affiliate_platform: string;       // e.g. 'amazon', 'rei' вЂ” for GTM data attribute only
  pros: string[];
  cons: string[];
  image: string;
}
```

**Renderer applies compliance attributes вЂ” GTM does the rest:**
```tsx
// ProductRenderer.tsx
<a
  href={product.affiliate_url}
  rel="nofollow sponsored"
  target="_blank"
  data-affiliate-platform={product.affiliate_platform}
>
  Buy on {product.affiliate_platform}
</a>
```

**GTM trigger:** Click trigger on elements with `data-affiliate-platform` attribute. No custom JS needed.

#### 2.2 Affiliate Partner Configuration
**Priority:** P0

All affiliate partners registered centrally. Renderers reference this config вЂ” never hardcode tracking IDs in content.

```typescript
// src/config/affiliates.ts
export const AFFILIATES = {
  amazon:     { tag: 'YOUR_TAG',  baseUrl: 'https://amazon.com' },
  rei:        { aid: 'YOUR_ID',   baseUrl: 'https://rei.com' },
  backcountry:{ aid: 'YOUR_ID',   baseUrl: 'https://backcountry.com' },
  moosejaw:   { aid: 'YOUR_ID',   baseUrl: 'https://moosejaw.com' },
};
```

**Primary partner:** Amazon Associates (backpacks/gear вЂ” broad catalog, instant approval)
**Secondary:** REI, Backcountry, Moosejaw (better commissions, niche authority)

#### 2.3 Affiliate Disclosure
**Priority:** P0 (Legal requirement)

Disclosure is rendered by the layout/renderer вЂ” never duplicated in content files.

- **Global:** Footer disclosure on all pages
- **Per-page:** Auto-rendered by `ResourcePage.astro` when any product in the JSON has an `affiliate_url`
- **Per-product:** Rendered inline by `ProductRenderer` next to each buy button

**Disclosure text:**
```
This page contains affiliate links. If you buy through them,
I earn a small commission at no extra cost to you.
I only recommend gear I've personally researched or used.
```

#### 2.4 Link Attribution Rules
**Priority:** P0

| Link Type | rel attribute | Handled by |
|-----------|--------------|------------|
| Affiliate links | `nofollow sponsored` | Renderer (automatic) |
| Editorial external | `noopener` | Renderer (automatic) |
| Internal | none | Renderer (automatic) |

No content author ever writes `rel` attributes manually вЂ” renderers enforce compliance at build time.

### 5. Content Types & Structure

#### 5.1 Content Architecture: JSON-First

**Core Principle:** Content and design are fully separated.
- **Content** = JSON files with structured data (AI-generated or human-authored)
- **Design** = Astro components that consume JSON and render HTML
- Redesigning a page never requires touching content files
- Each content type has its own JSON schema and dedicated renderer component

**Two content formats:**

| Format | Used For | Astro Collection Type |
|--------|----------|-----------------------|
| JSON | All structured/programmatic pages | `type: 'data'` |
| Markdown | Narrative blog posts only | `type: 'content'` |

**Content creation flow:**
```
AI pipeline в†’ JSON file (validated schema) в†’ Astro build в†’ Static HTML page
```

---

#### 5.2 Resource Pages (Bulk of site)
**Priority:** P0

Resource pages are the primary page type. They are fully static HTML, SEO-optimized, and generated from structured JSON. Each content type has its own schema and renderer.

**Content type groups:**

**Comparison Pages**
- Side-by-side gear/destination/service comparisons
- Rendered as structured HTML tables with affiliate links
- URL: `/compare/[item-a]-vs-[item-b]/`

**Buying Guides / Roundups**
- "Best X for Y" format (best carry-on backpacks, best hotels in Tokyo)
- Each item has affiliate link, pros/cons, recommendation context
- URL: `/guides/best-[category]/`

**Checklists**
- Packing lists, trip planning checklists
- Interactive checkboxes rendered client-side (JS island)
- URL: `/checklists/[topic]/`

**How-To Guides**
- Step-by-step instructional content
- Numbered steps with contextual affiliate links
- URL: `/guides/how-to-[topic]/`

**Glossaries**
- Travel term definitions with anchor links
- Highly linkable, long-tail keyword value
- URL: `/glossary/[topic]/`

**Statistics Pages**
- Curated travel data and stats with citations
- URL: `/statistics/[topic]/`

**Destination Resources**
- Practical info pages for destinations (costs, transport, visa)
- URL: `/destinations/[place]/[topic]/`

**Example JSON schema (buying guide):**
```typescript
interface BuyingGuide {
  meta: {
    content_type: 'buying-guide';
    category: string;
  };
  seo: {
    title: string;
    description: string;
    keywords: string[];
  };
  content: {
    intro: string;
    items: {
      name: string;
      affiliate_url: string;
      affiliate_platform: string;
      image: string;
      pros: string[];
      cons: string[];
      best_for: string;
      verdict: string;
    }[];
    faq: { question: string; answer: string; }[];
  };
}
```

#### 5.3 Tool Pages
**Priority:** P1

Tool pages are static HTML wrappers with one interactive component (Astro Island). Search engines index all surrounding content; the tool itself runs client-side only.

**Architecture:**
```astro
---
// Static wrapper вЂ” fully rendered HTML
import Calculator from '../components/tools/Calculator.tsx';
---

<h1>Freelance Rate Calculator</h1>        <!-- indexed by search engines -->
<p>Use this calculator to...</p>           <!-- indexed -->
<Calculator client:idle />                 <!-- hydrates when browser is idle -->
<section>How to use this tool...</section> <!-- indexed -->
```

**Hydration strategy:**
- `client:load` вЂ” tool is above the fold, needed immediately
- `client:idle` вЂ” tool is below the fold, loads when browser is idle
- `client:visible` вЂ” tool loads only when scrolled into view

**Tool types in scope:**
- **Calculators** вЂ” rate, budget, ROI, cost estimators
- **Filter Tables** вЂ” filterable/sortable data tables
- **Unit Converters** вЂ” currency, measurements, time zones
- **Quizzes / Assessments** вЂ” decision support flows
- **Word/Character Counters** вЂ” writing utilities

**URL structure:** `/tools/[tool-name]/`

### 5.7 Expansion Path

The architecture is designed for incremental complexity:

| Phase | Content Types | Tools |
|-------|--------------|-------|
| v1 | Buying guides, comparisons, basic checklists | None |
| v2 | How-to guides, glossaries, statistics, destination resources | Filter tables |

1. Define JSON schema
2. Add Astro `type: 'data'` collection
3. Build one renderer component
4. No changes to existing pages or content

## Technical Architecture

### Tech Stack

**Core:**
- **Framework:** Astro 4.x
- **Language:** TypeScript
- **CSS:** Tailwind CSS (or vanilla CSS)
- **Markdown:** MDX support
- **Interactive Islands:** React (via `@astrojs/react`) вЂ” only for tool components

- **Images:** Astro's built-in image optimization
- **Forms:** Formspree or Netlify Forms

**Integrations:**
- `@astrojs/sitemap`
- `@astrojs/rss`
- `@astrojs/mdx`
- `@astrojs/react` (for interactive tool islands)
- `@astrojs/tailwind` (optional)

**Analytics & Tracking:**
- **Google Tag Manager:** Centralized tag and event management
- **dataLayer:** Custom event tracking implementation
- **No direct GA4 integration:** All analytics routed through GTM

**Deployment:**
- **Hosting:** GitHub Pages
- **CI/CD:** GitHub Actions
- **Domain:** Custom domain + Cloudflare DNS

export function pushToDataLayer(eventData: GTMEvent) {
  if (typeof window !== 'undefined') {
    window.dataLayer = window.dataLayer || [];
    window.dataLayer.push(eventData);
  }
}
